Fehlercodes

Dieses Thema enthält eine Zusammenfassung der HTTP-Statuscodes für Zencoder-API-Anforderungen und eine Index der Fehlercodes wird von der Zencoder-API zurückgegeben. Klicken Sie auf eine der Fehlercodes um mehr Details zu sehen.

HTTP-Statuscodes

In der folgenden Tabelle ist der HTTP-Antwortcode für Zencoder-API-Anforderungen aufgeführt. Im Fehlerfall wird nach Möglichkeit die wahrscheinlichste Ursache angegeben.

HTTP-Antwortcodes
Statuscode Mögliche Erklärung
200 OK Standard-Erfolgsantwort für Aktionen, die kein neues Asset erstellen
201 CREATED Standardantwort für Anforderungen, die neue Assets erstellen
204 NO CONTENT Standardantwort für eine Anforderung, die keine Antwort zurückgibt
400 BAD REQUEST Ungültiger JSON im Anforderungshauptteil
401 UNAUTHORIZED Fehlender oder ungültiger API-Schlüssel
402 PAYMENT REQUIRED Wird zurückgegeben, wenn eine API-Anforderung versucht, den Zencoder-Integrationsmodus https://app.zencoder.com/docs/api/accounts/integration für ein Konto zu deaktivieren, für das keine Abrechnung eingerichtet wurde. Sie sollten auf www.zencoder.com gehen und einen Zahlungsplan wählen
403 FORBIDDEN Zinslimit für Anfragen, die überschritten wurden, oder als Reaktion auf eine erneute oder erneute Übermittlung einer API-Anforderung für Ressourcen, die zu oft wiederholt wurden
404 NOT FOUND Falsche Anforderungs-URL oder ungültige / alte Job-ID
406 NOT ACCEPTIBLE Wird zurückgegeben, wenn versucht wird, einen Nicht-AWS-Berechtigungsnachweis über die API zu erstellen
409 CONFLICT Im Allgemeinen nur, wenn versucht wird, einen zu alten Job erneut zu versuchen / erneut zu senden
422 UNPROCESSIBLE ENTITY Wird zurückgegeben, wenn eine API-Anforderung nicht validiert werden kann
500 INTERNAL SERVER ERROR Entweder ein vorübergehendes Problem mit Zencoder oder ein Fehler mit der Anfrage
503 SERVICE UNAVAILABLE Zencoder vorübergehend nicht verfügbar, wahrscheinlich während der geplanten Wartung
504 GATEWAY TIMEOUT Wird zurückgegeben, wenn der Zencoder-Dienst nicht verfügbar ist
