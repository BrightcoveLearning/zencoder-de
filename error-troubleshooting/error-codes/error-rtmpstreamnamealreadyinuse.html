Bedeutung

Dieser Fehler bedeutet, dass das CDN einen RTMP-Stream abgelehnt hat, da der Streamname bereits von einem anderen Stream verwendet wird. Mit anderen Worten, zwei RTMP-Encoder versuchen, auf denselben CDN-Einstiegspunkt zu pushen, und der zweite wird abgelehnt, weil der erste bereits ausgeführt wird.

Mehr Informationen

Dies kann bedeuten:

  • Ein anderer Zencoder-Job wird bereits auf diesen CDN-Stream-Namen gestreamt
  • Der CDN-Stream-Name wird von einem anderen System außerhalb von Zencoder verwendet
  • Ein früherer Job konnte aus irgendeinem Grund nicht vom CDN getrennt werden

Um dieses Problem zu vermeiden, können Sie einfach einen anderen CDN-Stream-Namen verwenden oder frühere Jobs mit demselben Stream-Namen beenden. Wenn ein Job aus irgendeinem Grund nicht beendet werden kann, Kontaktieren Sie Support mit der Job ID und unser Engineering-Team wird untersuchen.

Diesen Job wiederholen?

Jobs mit diesem Fehler sollte nicht erneut versucht werden. Erfahren Sie mehr über das Wiederholen von Jobs.
