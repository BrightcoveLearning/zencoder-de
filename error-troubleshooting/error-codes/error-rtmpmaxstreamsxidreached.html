Bedeutung

Dieser Fehler bedeutet, dass das CDN einen RTMP-Stream abgelehnt hat, da zu viele Streams dieselbe Stream-ID verwenden. Die maximale Anzahl von Streams pro ID wird also überschritten.

Mehr Informationen

Dies kann bedeuten:

  • Die maximale Anzahl von Streams pro ID, die im CDN konfiguriert sind, ist zu niedrig, um alle angeforderten Streams zu akzeptieren
  • Andere Zencoder-Jobs werden bereits auf dieses CDN und diese Stream-ID gestreamt
  • Die CDN-Stream-ID wird von einem anderen System außerhalb von Zencoder verwendet
  • Das CDN könnte „' Zombie-Streams haben, bei denen ein vorheriger Stream nicht richtig getrennt wurde und immer noch auf das maximale Stream-Limit für diese ID zählt

Um dieses Problem zu vermeiden, können Sie die anderen Jobs mit Streams mit dieser Stream-ID stoppen oder eine andere Stream-ID verwenden, wenn Sie diese haben. Für weitere Informationen können Sie Kontaktieren Sie Support mit der Job ID und unser Engineering-Team wird untersuchen.

Diesen Job wiederholen?

Jobs mit diesem Fehler sollte nicht erneut versucht werden. Erfahren Sie mehr über das Wiederholen von Jobs.
