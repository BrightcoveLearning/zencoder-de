Einen API-Schlüssel erhalten

Alle Anforderungen an die Zencoder-API müssen mithilfe eines API-Schlüssels authentifiziert werden. In diesem Thema werden die verschiedenen Arten von API-Schlüsseln für Zencoder und deren Beschaffung erläutert.

Einen API-Schlüssel erhalten

Um einen API-Schlüssel anzufordern, gehen Sie zu die Zencoder-Site. Wenn Sie sich bereits als Benutzer registriert haben, melden Sie sich an. Wenn nicht, registrieren Sie sich als neuer Benutzer. Sie können API-Schlüssel erst anfordern, wenn Sie als Benutzer registriert sind.

Sobald Sie angemeldet sind, gehen Sie zum API-Abschnitt:

Im API-Abschnitt sehen Sie Optionen zum Anfordern verschiedener Arten von API-Schlüsseln:

In den folgenden Abschnitten finden Sie eine Erläuterung der verschiedenen Arten von API-Schlüsseln. Beachten Sie, dass Sie für einen kostenpflichtigen Tarif angemeldet sein müssen, um Vollzugriffsschlüssel zu erhalten.

Vollzugriffsschlüssel

Mit Vollzugriffsschlüsseln können Sie jede Art von Anforderung an die Zencoder-API senden. Dies ist die Art von Schlüssel, die Sie benötigen, um Aufträge an Zencoder zu senden und Ausgaben zu erhalten.

Schreibgeschützte Schlüssel

Mit schreibgeschützten Zugriffsschlüsseln können Sie erstellen GET Anfragen nur an die Zencoder-API. Diese sind nützlich für Benutzer, die nur den Status von Jobs abrufen müssen, aber keine Jobs senden. Sehen Verwenden schreibgeschützter API-Schlüssel für mehr Informationen.

Schlüssel für den Integrationsmodus

Mit den Schlüsseln für den Integrationsmodus können Sie Jobs senden, die Jobs werden jedoch nicht verarbeitet. Sie sind vor allem für Systemintegratoren nützlich, die Fehleranforderungen testen möchten. Sie können auch verwendet werden, um ein Gefühl für die Verwendung der API zu bekommen, bevor Sie einen bezahlten Zencoder-Plan kaufen, oder um Anforderungen zu überprüfen, bevor Sie tatsächlich Ausgaben erhalten.
