    Video-Einstellungen

    Dieses Thema enthält Details zu den Codierungseinstellungen in Bezug auf allgemeine Einstellungen für Videoausgänge.

    video_bit_depth

    video_bit_depth:Number

    API-Versionen: V2

    Elternteil: outputs

    Standard: 10

    Gültige Werte: 8, 10 oder 12

    Kompatible Job-Typen: vod, live

    Beispiel: 5

    Beschreibung:

    Die Anzahl der Bits pro Farbkanal.

    Normalerweise würden Sie dies nicht einstellen und stattdessen das angeben video_codec_profile das würde die Bittiefe liefern.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_bit_depth": 10
    }
  ]
}

    video_reference_frames

    video_reference_frames:String

    API-Versionen: V2

    Elternteil: outputs

    Standard: 3

    Gültige Werte: 0-16, auto

    Kompatible Job-Typen: vod, live

    Beispiel: 5

    Beschreibung:

    Anzahl der zu verwendenden Referenzframes.

    Mehr Referenzrahmen führen zu einer etwas höheren Komprimierungsqualität, aber einer erhöhten Decodierungskomplexität. In der Praxis hat es selten viel Nutzen, über 5 zu gehen.

    Teilweise bestimmt durch speed ebenso gut wie video_codec_profile. Stellen Sie "auto" ein, um unsere zuzulassen speed Einstellung, um diese Nummer natürlich zu wählen.

    Wir verwenden standardmäßig 3 als einen guten Kompromiss zwischen Komprimierung und Decodierungskomplexität.

    Verwenden Sie 1 für Videos, die für ältere iPod- oder iPhone-Videos der ersten Generation erstellt wurden, oder für andere technisch eingeschränkte Decoder.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_reference_frames": 5
    }
  ]
}

    video_codec_profile

    video_codec_profile:String

    API-Versionen: V2

    Elternteil: outputs

    Standard: H.264: baseline , HEVC: main , VP9: 0

    Gültige Werte: Codec-abhängig

    Kompatible Job-Typen: Alle

    Beispiel: hoch

    Beschreibung:

    Legt das vom Video-Codec verwendete Codierungsprofil fest.

    • H.264 Profile: baseline , main , high , high10 , high422 , high444.
    • HEVC-Profile: main , main10 , main12 , main422-10 , main422-12 , main444-8 , main444-10 , main444-12.
    • VP9-Profile: 0, 1, 2, 3.

    Im Allgemeinen werden niedrigere Ebenen von einer größeren Anzahl von Geräten unterstützt und sind möglicherweise einfacher zu dekodieren. Höhere Ebenen bieten mehr Funktionen wie zusätzliche Bittiefen und Auswahlmöglichkeiten für die Chroma-Unterabtastung. In einigen Fällen unterstützen höhere Werte auch eine erhöhte Komprimierungseffizienz.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_codec_profile": "high"
    }
  ]
}

    video_codec_level

    video_codec_level:String

    API-Versionen: V2

    Elternteil: outputs

    Standard: Wird automatisch anhand eines auf dem Ausgabevideo basierenden Ebenendiagramms berechnet. Sehen H.264-Ebenen , HEVC-Level

    Gültige Werte: Ein gültiges Level: H.264 Ebenen , HEVC-Werte.

    Kompatible Job-Typen: Alle

    Beispiel: 4.1

    Beschreibung:

    Beschränkt die Bitrate, Makroblöcke (H.264) oder die Bitrate und codiert Baumeinheiten (HEVC).

    Wird hauptsächlich für die Gerätekompatibilität verwendet. Das iPhone unterstützt beispielsweise H.264 Level 3, dh ein Video decoder_bitrate_cap darf 10.000 kbps nicht überschreiten. In der Regel sollten Sie diese Einstellung nur ändern, wenn Sie auf ein bestimmtes Gerät abzielen, für das dies erforderlich ist.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_codec_level": "4.1"
    }
  ]
}

    video_bframes

    video_bframes:Integer

    API-Versionen: V2

    Elternteil: outputs

    Standard: 0

    Gültige Werte: 0-16

    Kompatible Job-Typen: Alle

    Beispiel: 2

    Beschreibung:

    Maximale Anzahl aufeinanderfolgender B-Frames.

    Die Verwendung von B-Frames kann die Qualität bei derselben Bitrate erhöhen, ist jedoch anspruchsvoller für CPUs. Dies kann auf alten Computern oder Geräten problematisch sein, funktioniert jedoch auf moderner Hardware im Allgemeinen reibungslos.

    H.264 unterstützt durch B-Frames mit Main or High profile , nicht Baseline. Der Standardwert ist 0, um die größtmögliche Kompatibilität zu gewährleisten.

    Für HEVC können B-Frames auf jeder Ebene verwendet werden. Der Standardwert ist 3.

    Wir empfehlen einen Wert von 3 für Verbesserungen der Kompressions-/Qualität. Werte über 5 oder 6 bieten selten große Vorteile und verlängern die Codierungszeit erheblich.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_bframes": 2
    }
  ]
}

    Siehe auch: video_codec_profile

    color_metadata

    color_metadata:String

    API-Versionen: V2

    Elternteil: outputs

    Standard: preserve

    Gültige Werte: preserve oder discard

    Kompatible Job-Typen: Alle

    Beispiel: discard

    Beschreibung:

    Diese Option wird derzeit auf H.264-, HEVC- und VP9-Ausgängen unterstützt. Wenn das Eingangsvideo Matrixkoeffizienten, Übertragungseigenschaften und Farbprimärmetadaten enthält, wird Zencoder dies tun preserve diese Einstellungen standardmäßig.

    Legen Sie fest, dass diese Metadaten nicht in Ausgaben eingebettet sind color_metadata zu discard.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "color_metadata": "discard"
    }
  ]
}
