Überblick

Beim Transmuxing wird das Format einer Video- oder Audiodatei geändert, wobei einige oder alle Streams aus der Originaldatei beibehalten werden.

Heute kann Zencoder H.264-Video- und AAC/MP3-Audiospuren von MP4/MOV-Formaten in MP4/TS umwandeln (einschließlich der Erstellung fragmentierter MP4- oder segmentierter TS-Ausgabe). Es gibt jedoch eine Reihe von Anforderungen, die von den Eingabedateien erfüllt werden müssen, damit Transmux ordnungsgemäß ausgeführt werden kann:

Video

  • MP4/MOV-Containerformat
  • H.264 Codec
  • Keine B-Frames verwendet

Audio

  • MP4/MOV-Containerformat
  • AAC- oder MP3-Codec

Segmentieren / Fragmentieren

  • Die maximale GOP sollte kleiner oder gleich der Segmentdauer sein
  • HE-AAC-V2 darf nicht verwendet werden

Andere

  • Die meisten Codierungsoptionen haben keine Auswirkungen auf transmuxierte Dateien, da die Streams nicht neu codiert werden

copy_video

copy_video:Boolean

API-Versionen: V1, V2

Elternteil: Ausgabe

Standard: falsch

Gültige Werte: richtig oder falsch

Kompatible Auftragstypen: Alle

Beispiel: wahr

Beschreibung:

Angeben copy_video nimmt die Videospur aus der Eingabevideodatei und überträgt sie in die resultierende Ausgabedatei.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "copy_video": true
    }
  ]
}

copy_audio

copy_audio:Boolean

API-Versionen: V1, V2

Elternteil: Ausgabe

Standard: falsch

Gültige Werte: richtig oder falsch

Kompatible Auftragstypen: Alle

Beispiel: wahr

Beschreibung:

Angeben copy_audio nimmt die Audiospur aus der Eingangsvideodatei und überträgt sie in die resultierende Ausgabedatei.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "copy_audio": true
    }
  ]
}
