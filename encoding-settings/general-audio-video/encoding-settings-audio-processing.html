audio_gain

audio_gain:Float

API-Versionen: V1, V2

Elternteil: Ausgänge

Gültige Werte: Eine Zahl zwischen -60 und 60

Kompatible Auftragstypen: Alle

Beispiel: 5

Beschreibung:

Wenden Sie einen Verstärkungsbetrag auf das Audio an. Dies erhöht oder verringert die Lautstärke des Audios um den angegebenen Betrag, nachdem andere Effekte angewendet wurden. Dies ist hauptsächlich nützlich, um die Pegel nach dem Anwenden anderer Effekte anzupassen oder um Anpassungen an Dateien vorzunehmen, für die Sie den Pegel bereits kennen. Klänge, die nur ein wenig über die maximale Lautstärke hinausgehen, werden sanft begrenzt, um die Wiedergabetreue zu erhalten. Wenn Sie jedoch die maximale Lautstärke deutlich erhöhen, wird der Klang verzerrt. Sehen audio_normalize zum automatischen Erhöhen der Lautstärke ohne Verzerrung. Angegeben in dB, entweder positiv oder negativ, bis zu 60 dB.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_gain": 5
    }
  ]
}

Siehe auch: audio_normalize

audio_normalize

audio_normalize:Boolean

API-Versionen: V1, V2

Elternteil: Ausgänge

Standard: falsch

Gültige Werte: richtig oder falsch

Kompatible Auftragstypen: vod

Beispiel: wahr

Beschreibung:

Normalisieren Sie Audio auf 0 dB. Dies erhöht die Lautstärke des Audios so weit wie möglich, ohne Verzerrungen zu verursachen. Um eine konsistente Anpassung durch die Expansions- und Komprimierungseffekte zu gewährleisten, normalisieren wir die Lautstärke sowohl vor als auch nach diesen Effekten. Sehen audio_pre_normalize und audio_post_normalize nur an einem dieser Punkte zu normalisieren. audio_normalize entspricht der Angabe von audio_pre_normalize und audio_post_normalize.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_normalize": true
    }
  ]
}

Siehe auch: audio_gain, audio_pre_normalize und audio_post_normalize

audio_highpass

audio_highpass:Integer

API-Versionen: V1, V2

Elternteil: Ausgänge

Gültige Werte: Eine Frequenz in Hz von 5 bis 24000

Kompatible Auftragstypen: Alle

Beispiel: 10

Beschreibung:

Wenden Sie einen Hochpassfilter auf das Audio mit der angegebenen Frequenz in Hz an. Dadurch wird verhindert, dass Audiofrequenzen unterhalb des angegebenen Schwellenwerts in die Ausgabe aufgenommen werden. Eine sehr häufige Verwendung dieser Einstellung ist das Reduzieren von Rumpeln / Geräuschen durch Audio, wie z. B. die Geräusche von Lastwagen, die während der Aufnahme draußen vorbeifahren, oder einige Geräusche, die durch Wind oder Schlagen eines Mikrofons verursacht werden. DC-Shift kann auch reduziert / entfernt werden, indem ein Hochpassfilter mit einer niedrigen Frequenz wie 10 Hz angegeben wird.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_highpass": 10
    }
  ]
}

Siehe auch: audio_lowpass

audio_compression_ratio

audio_compression_ratio:Float

API-Versionen: V1, V2

Elternteil: Ausgänge

Gültige Werte: 1,0 bis 30,0

Kompatible Auftragstypen: Alle

Beispiel: 2.5

Beschreibung:

Komprimieren Sie den Dynamikbereich des Audios, indem Sie die Lautstärke in einem festgelegten Verhältnis von N: 1 über dem angegebenen verringern audio_compression_threshold. Verhältnisse zwischen 1,0 und 30,0 sind gültig. Die Komprimierung ist im Wesentlichen nützlich, um die Lautstärke zwischen leisen und lauten Punkten gleichmäßiger zu gestalten. Dies kann verwendet werden, um laute Stellen (Schreien oder Zuschlagen von Türen usw.) weniger störend zu machen. Wird in Verbindung mit verwendet audio_gain oder audio_normalization kann es leise Teile einer Sprache hörbarer machen.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_compression_ratio": 2.5
    }
  ]
}

Siehe auch: audio_compression_threshold

audio_karaoke_mode

audio_karaoke_mode:Boolean

API-Versionen: V1, V2

Elternteil: Ausgänge

Standard: falsch

Gültige Werte: richtig oder falsch

Kompatible Auftragstypen: Alle

Beispiel: wahr

Beschreibung:

Wenden Sie einen "Karaoke" -Effekt auf das Audio an, indem Sie den rechten Audiokanal vom linken subtrahieren und nur den Unterschied beibehalten. Gilt nur für Stereoausgänge.

Seien Sie vorsichtig mit dieser Funktion:

  • Es erzeugt manchmal Verzerrungen
  • Es macht manchmal überhaupt nichts
  • Es wirkt sich manchmal negativ auf Hintergrundgeräusche oder Musik aus
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_karaoke_mode": true
    }
  ]
}
