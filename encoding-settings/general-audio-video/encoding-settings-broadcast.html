Überblick

Zencoder bietet Unterstützung für erweiterte Codierungsformate und Codecs für Broadcast- und professionelle Workflows.

video_codec_preset

video_codec_preset:String

API-Versionen: V2

Elternteil: outputs

Gültige Werte: avci_50, avci_100

Kompatible Auftragstypen: Alle

Beispiel: avci_100

Beschreibung:

Voreinstellung für die Videokodierung, um die Kompatibilität mit bestimmten Industriestandards zu ermöglichen.

Standards:

  • avci_50 ist eine reine AVC 10-Bit-Intra-Frame-Komprimierung. 50 Mbit/s, Hoch 10 Intra-Profil, Stufe 3.2, 4. Siehe SMTP RP 2027: 2007.
  • avci_100 ist eine reine AVC 10-Bit-Intra-Frame-Komprimierung. 100 Mbit/s, Hoch 4:2:2 Intra-Profil, Stufe 4.1. Siehe SMTP RP 2027: 2007.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_codec_preset": "avci_100",
      "video_bit_depth": 10
    }
  ]
}

format_preset

format_preset:String

API-Versionen: V2

Elternteil: outputs

Gültige Werte: ts_dvb, ts_cablelabs, ts_broadcast, as11_hd

Kompatible Auftragstypen: Alle

Beispiel: ts_broadcast

Beschreibung:

Format voreingestellt, um Kompatibilität mit bestimmten Industriestandards zu ermöglichen. Beachten Sie, dass nur 2 Audiokanäle unterstützt werden.

Standards:

  • ts_broadcast erzeugt standardmäßige MPEG-TS-Broadcast-kompatible Streams.
  • ts_dvb erzeugt DVB-kompatible Streams (Digital Video Broadcasting).
  • ts_cablelabs erzeugt CableLabs-kompatible MPEG-TS-Streams.
  • as11_hd ist AVC-Intra 100-Video, 4 oder 16 Kanäle PCM-Audio in einem MXF-OP1A-Container.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "format_preset": "ts_broadcast"
    }
  ]
}

Siehe auch: Format

ts_muxrate

ts_muxrate:Integer

API-Versionen: V2

Elternteil: outputs

Gültige Werte: Eine positive ganze Zahl. 1000000 max.

Kompatible Auftragstypen: Alle

Beispiel: 2600

Beschreibung:

Die Höchstrate in Kbit/s, dass der Transportstrom muxiert werden kann. Wird nur mit dem ts-Format verwendet.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "format_preset": "ts_broadcast",
      "ts_muxrate": 2600,
      "video_bitrate": 2000,
      "audio_bitrate": 128,
      "audio_sample_rate": 48000,
      "audio_constant_bitrate": true
    }
  ]
}

Siehe auch: format_preset

ts_cbr

ts_cbr:Boolean

API-Versionen: V2

Elternteil: outputs

Standard: falsch

Gültige Werte: richtig oder falsch

Kompatible Auftragstypen: Alle

Beispiel: wahr

Beschreibung:

Erstellen Sie eine CBR-Ausgabe, indem Sie Pakete auf der Transportstromschicht stopfen.

Dies ist für vollständig kompatible Broadcast-Streams erforderlich, kann jedoch erheblich größere Dateien erstellen. Nur zur Verwendung mit Sendungen.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "format_preset": "ts_broadcast",
      "ts_cbr": true,
      "video_bitrate": 2000,
      "audio_bitrate": 128,
      "audio_sample_rate": 48000,
      "audio_constant_bitrate": true
    }
  ]
}

Siehe auch: format_preset

verlustfreies_video

lossless_video:Boolean

API-Versionen: V2

Elternteil: outputs

Standard: falsch

Gültige Werte: richtig oder falsch

Kompatible Auftragstypen: Alle

Beispiel: wahr

Beschreibung:

Aktiviert verlustfreie Videokodierung, wenn dies vom Videocodec unterstützt wird.

Derzeit nur von Jpeg2000 (jp2) unterstützt.

Kann nicht mit verwendet werden video_bitrate.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_codec": "jp2",
      "lossless_video": true
    }
  ]
}

Siehe auch: video_bitrate
