min_size

min_size:String

API-Versionen: V2

Elternteil: bedingte

Kompatible Auftragstypen: Alle

Beispiel:

  • 640 x 480

Beschreibung:

Wenn die Abmessungen der Quelle kleiner als die angeforderte Größe sind, wird die Ausgabe übersprungen.

Die Abmessungen werden berechnet, indem die Höhe und die Breite der Quelle multipliziert und mit dem Produkt der angegebenen Abmessungen verglichen werden.

Hinweis: Aufgrund der Art der Videometadaten und der Inkonsistenzen zwischen den Videotypen können die Abmessungen einiger Quelldateien während der Überprüfung falsch gelesen werden, was dazu führt, dass Bedingungen falsch angewendet oder ignoriert werden.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "min_size": "640x480"
      }
    }
  ]
}

min_dauer

min_duration:Integer

API-Versionen: V2

Elternteil: bedingte

Gültige Werte: Eine positive ganze Zahl

Kompatible Auftragstypen: Alle

Beispiel:

  • 300

Beschreibung:

Wenn die Dauer der Quelle kürzer als die angeforderte Dauer ist, wird die Ausgabe übersprungen.

Die Dauer muss in Sekunden angegeben werden. Verwenden Sie '300', um eine Ausgabe zu überspringen, wenn die Quelle kürzer als 5 Minuten ist.

      }
    }
  ]

min_audio_bitrate

min_audio_bitrate:Integer

API-Versionen: V2

Elternteil: bedingte

Gültige Werte: Eine positive ganze Zahl

Kompatible Auftragstypen: Alle

Beispiel:

  • 128

Beschreibung:

Wenn die Audio-Bitrate der Quelle unter der angegebenen Bitrate (in kbit / s) liegt, wird die Ausgabe übersprungen.

Quelldateien ohne Audiospur oder ohne erkennbare Audiobitrate ignorieren diese Einstellung. (Sehen require_audio für mehr Optionen.) Verwenden Sie 128, um eine Ausgabe zu überspringen, wenn die Quell-Audio-Bitrate weniger als 128 Kbit / s beträgt.

Hinweis: Aufgrund der Art der Audio-Metadaten und der Inkonsistenzen zwischen den Audiotypen werden bei einigen Quelldateien die Bitraten möglicherweise während der Überprüfung falsch gelesen, was dazu führt, dass Bedingungen falsch angewendet oder ignoriert werden.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "min_audio_bitrate": 128
      }
    }
  ]
}

Siehe auch: max_audio_bitrate und require_audio

require_audio

require_audio:Boolean

API-Versionen: V2

Elternteil: bedingte

Standard: falsch

Gültige Werte: richtig oder falsch

Kompatible Auftragstypen: Alle

Beispiel: true

Beschreibung:

Wenn die Quelldatei keine Audiospur enthält, wird die Ausgabe übersprungen.

Um eine Ausgabe zu überspringen, wenn die Quelle keine Audiospur hat, setzen Sie diese Option auf true.

Hinweis: Aufgrund der Art der Medienmetadaten und der Inkonsistenzen zwischen den Medientypen werden bei einigen Quelldateien die Metadaten möglicherweise während der Überprüfung falsch gelesen, was dazu führt, dass Bedingungen falsch angewendet oder ignoriert werden.


  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "require_audio": true,
        "min_audio_bitrate": 64
      }
    }
  ]
}

Siehe auch: min_audio_bitrate und require_video
