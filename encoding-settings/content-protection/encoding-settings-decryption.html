Überblick

Zencoder unterstützt die Verarbeitung verschlüsselter Mediendateien mithilfe des Paradigmas „Verschlüsselung in Ruhe“. Wir rufen die verschlüsselten Dateien von Ihrem Server ab und entschlüsseln sie nur während der Verarbeitung. Die entschlüsselte Version der Datei wird sofort nach der Verarbeitung entfernt, um Ihre Daten zu schützen.

Wir unterstützen sowohl AES-128- als auch AES-256-verschlüsselte Dateien im CBC- und CTR-Modus. Entschlüsselungsparameter müssen in Ihrer Jobanfrage enthalten sein. Die Entschlüsselung ist aktiviert, wenn entweder a decryption_key oder decryption_key_url angegeben.

decryption_method

decryption_method:String

API-Versionen: V1, V2

Elternteil: Eingang

Standard: aes-128-cbc (wenn decryption_key oder decryption_key_url eingestellt sind)

Gültige Werte: "keine", "aes-128-cbc", "aes-128-ctr", "aes-256-cbc" oder "aes-256-ctr"

Kompatible Auftragstypen: vod

Beispiel: aes-128-cbc

Beschreibung:

Stellen Sie den Entschlüsselungsalgorithmus ein, der zum Entschlüsseln verwendet werden soll.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_method": "aes-128-cbc"
  }

decryption_key

decryption_key:String

API-Versionen: V1, V2

Elternteil: Eingang

Gültige Werte: Eine hexadezimale Zeichenfolge mit 16 Oktetten (32 Zeichen lang, optionales "0x" -Präfix)

Kompatible Auftragstypen: vod

Beispiel: 1234567890abcdef1234567890abcdef

Beschreibung:

Legen Sie den Entschlüsselungsschlüssel fest, der für eine verschlüsselte Eingabe verwendet werden soll.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_key": "1234567890abcdef1234567890abcdef"
  }

decryption_key_url

decryption_key_url:String

API-Versionen: V1, V2

Elternteil: Eingang

Gültige Werte: Eine URL zu einer Datei, die einen 16-Byte-Binärschlüssel enthält

Kompatible Auftragstypen: vod

Beispiel: http://example.com/encryption.key

Beschreibung:

Die URL einer zu verwendenden Entschlüsselungsschlüsseldatei mit einem Schlüssel in Binärform (insgesamt 16 Byte). Dies wird ignoriert, wenn ein Entschlüsselungsschlüssel angegeben wird. 
  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_key_url": "http://example.com/encryption.key"
  }

decryption_password

decryption_password:String

API-Versionen: V1, V2

Elternteil: Eingang

Gültige Werte: Ein Faden

Kompatible Auftragstypen: vod

Beispiel: Super-Secret-Passwort

Beschreibung:

Das Kennwort, das in Kombination mit dem Schlüssel zum Entschlüsseln der Eingabedatei verwendet wird.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_password": "super-secret-password"
  }
