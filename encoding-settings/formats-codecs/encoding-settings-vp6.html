    VP6-Einstellungen

    Dieses Thema enthält Details zu Codierungseinstellungen in Bezug auf VP6-Ausgänge.

    vp6_temporal_down_watermark

    vp6_temporal_down_watermark:Number

    API-Versionen: V1, V2

    Elternteil: Ausgänge

    Standard: 20

    Gültige Werte: 0-100

    Kompatible Job-Typen: Alle

    Beispiel: 10

    Beschreibung:

    Codec-Parameter für den Prozentsatz des zeitlichen Wasserzeichens nach unten. Gibt den Prozentsatz des verbleibenden Dataratepuffers an, unter dem der Encoder Frames ablegen darf. Wird nur verwendet, wenn vp6_temporal_resampling aktiviert. Ein größerer Prozentsatz erhöht die Wahrscheinlichkeit, dass Frames gelöscht werden, um die angeforderte Videobitrate zu erreichen. Das Gegenteil ist auch der Fall.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_temporal_down_watermark": 10
    }
  ]
}

    Siehe auch: vp6_temporal_resampling

    vp6_temporal_resampling

    vp6_temporal_resampling:Boolean

    API-Versionen: V1, V2

    Elternteil: Ausgänge

    Standard: True beim Codieren mit einer geringen Anzahl von Bits pro Pixel. Ansonsten falsch.

    Gültige Werte: richtig oder falsch

    Kompatible Job-Typen: Alle

    Beispiel: wahr

    Beschreibung:

    Codec-Parameter für temporales Resampling. In besonders schwierigen Regionen wird der Encoder, wenn er aktiviert ist, Frames fallen, um das Ziel der Datenrate zu erreichen. Der Wert von vp6_temporal_down_watermark bestimmt den Prozentsatz des Dataratepuffers, unter dem der Encoder aktiviert ist, Frames abzulegen.

    Die Standardeinstellung ist "true" für Dateien mit einer geringen Anzahl von Bits pro Pixel und "false" für Dateien mit einer hohen Anzahl von Bits pro Pixel. Teilen Sie zur Berechnung die Video-Bitrate in Bits pro Sekunde von Breite * * Höhe * * frame_rate. Wenn diese Zahl unter 0,07 liegt, wird vp6_temporal_resampling standardmäßig auf true gesetzt. Beispiel: 600.000 Bit pro Sekunde (ca. 600 kbps) geteilt durch (640 x 480 * 30 fps) = 0,065. In diesem Fall wäre also vp6_temporal_resampling aktiviert.

    Hinweis: bei Verwendung Konstante Bit-Rate Im Modus wird diese Option auf true gesetzt. Einstellen vp6_temporal_down_watermark Ergebnisse anpassen.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_temporal_resampling": true
    }
  ]
}

    Siehe auch: vp6_temporal_down_watermark

    vp6_undershoot_pct

    vp6_undershoot_pct:Number

    API-Versionen: V1, V2

    Elternteil: Ausgänge

    Standard: 90

    Gültige Werte: 0-100

    Kompatible Job-Typen: Alle

    Beispiel: 75

    Beschreibung:

    Erstellt eine Ausgabe, die auf eine etwas niedrigere Datenrate abzielt, sodass im Puffer Bits zur Verfügung stehen, um schwierige Abschnitte zu verbessern. Dies ist ein Prozentsatz, also sind gültige Werte zwischen 0 und 100.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_undershoot_pct": 75
    }
  ]
}

    vp6_profile

    vp6_profile:String

    API-Versionen: V1, V2

    Elternteil: Ausgänge

    Standard: vp6e

    Gültige Werte: vp6s oder vp6e

    Kompatible Job-Typen: Alle

    Beispiel: vp6s

    Beschreibung:

    Entweder "vp6s" oder "vp6e". vp6e ist für den allgemeinen Gebrauch konzipiert. vp6s ist für Videos mit hoher Auflösung und Datenrate optimiert, indem die Komplexität der Decodierung verringert wird, was die Wiedergabe auf Computern mit eingeschränkten Ressourcen ermöglicht.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_profile": "vp6s"
    }
  ]
}

    vp6_compression_mode

    vp6_compression_mode:String

    API-Versionen: V1, V2

    Elternteil: Ausgänge

    Standard: gut

    Gültige Werte: gut oder am besten

    Kompatible Job-Typen: Alle

    Beispiel: Beste

    Beschreibung:

    Beeinflusst die Geschwindigkeit der Codierung im Vergleich zur Ausgabequalität. „gut“ gleicht Geschwindigkeit und Qualität aus. „best“ priorisiert Qualität gegenüber Geschwindigkeit, was zu höherer Komprimierung, aber langsamerer Codierung führt. Wenn Sie einstellen Geschwindigkeit bis 1 oder 2 und VP6 verwenden, verwenden wir den "besten" Modus; Andernfalls verwenden wir "gut".

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_compression_mode": "best"
    }
  ]
}

    Siehe auch: Geschwindigkeit

    vp6_2pass_min_section

    vp6_2pass_min_section:Integer

    API-Versionen: V1, V2

    Elternteil: Ausgänge

    Standard: 40

    Gültige Werte: 0-100

    Kompatible Job-Typen: Alle

    Beispiel: 50

    Beschreibung:

    Bei der VBR-Codierung mit zwei Durchgang steuert dies den niedrigsten Datenarat, den der Encoder für jeden Abschnitt zulässt, unabhängig davon, wie einfach der Abschnitt ist. Dieser Wert wird verwendet, um zu verhindern, dass schwierige Abschnitte zu viele Bits aus einfachen Abschnitten stehlen. Ausgedrückt als Prozentsatz, als Ganzzahl.

    Zum Beispiel wenn Video-Bitrate ist auf 1000 kbps und vp6_2_pass_min_section auf 50 eingestellt, die niedrigste Bitrate eines Abschnitts beträgt 500 kbps (1000 * 50%).

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_2pass_min_section": 50
    }
  ]
}

    Siehe auch: vp6_2pass_max_section

    vp6_2pass_max_section

    vp6_2pass_max_section:Integer

    API-Versionen: V1, V2

    Elternteil: Ausgänge

    Standard: 400

    Gültige Werte: Eine positive Ganzzahl

    Kompatible Job-Typen: Alle

    Beispiel: 250

    Beschreibung:

    Bei der VBR-Codierung mit zwei Durchwegen steuert dies das höchste Datenarat, das gestreamt werden kann, und auch das höchste Datenarat, das der Encoder zulässt, unabhängig davon, wie schwierig der Abschnitt ist.

    Zum Beispiel wenn Video-Bitrate ist auf 1000 kbps und vp6_2_pass_max_section auf 450 eingestellt, die höchste Bitrate eines Abschnitts beträgt 4500 kbps (1000 * 450%).

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_2pass_max_section": 250
    }
  ]
}

    Siehe auch: vp6_2pass_min_section

    vp6_denoise_level

    vp6_denoise_level:Float

    API-Versionen: V1, V2

    Elternteil: Ausgänge

    Standard: 0

    Gültige Werte: Dezimalzahl zwischen 0 und 1

    Kompatible Job-Typen: Alle

    Beispiel: 0.1

    Beschreibung:

    Legt den Geräuschpegel der Quelle fest. Jeder Wert größer als 0 impliziert einen konstanten und bekannten Rauschpegel innerhalb der Quelle. Ein Wert von 0 veranlasst den Filter, den Geräuschpegel zu schätzen, sodass er von Frame zu Frame variieren kann. Größere Geräuschpegel zeigen eine lautere Quelle an, die zu einer erhöhten Entfernung durch den Filter auf Kosten der Schärfe führt.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "vp6_denoise_level": 0.1
    }
  ]
}

    Siehe auch: entrauschen

    Konstante Bit-Rate

    constant_bitrate:Boolean

    API-Versionen: V1, V2

    Elternteil: Ausgänge

    Standard: false

    Gültige Werte: richtig oder falsch

    Kompatible Job-Typen: Alle

    Beispiel: wahr

    Beschreibung:

    Die CBR-Codierung (Constant Bitrate) wird von einigen Streaming-Servern und Decodern verwendet. Die CBR-Codierung sieht erheblich schlechter aus als die VBR-Codierung (Variable Bitrate) bei gleicher Bitrate. Verwenden Sie diese Einstellung daher nur, wenn Sie dies unbedingt müssen. Diese Einstellung erfordert, dass Sie a einstellen Video-Bitrate ;; Sie können nicht verwenden Qualität um eine Bitrate automatisch auszuwählen, wenn die Einstellung constante_bitrate verwendet wird.

    Verwenden Sie constant_bitrate nicht für die HTTP-basierte Bereitstellung oder für heruntergeladene Videos.

    Unterschiedliche video_codecs verhalten sich bei der Einstellung constante_bitrate unterschiedlich.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "constant_bitrate": true
    }
  ]
}
