HEVC/H.265 Überblick

HEVC (auch allgemein als H.265 bezeichnet) steht allen Kunden zur Verfügung. HEVC ist ein Videocodec der nächsten Generation, der in der Lage ist, Inhalte höherer Qualität bei niedrigeren Bitraten als sein Vorgänger H.264 bereitzustellen.

Kodierungsempfehlungen

video_codec_level: schränkt die Bitrate- und Codierbaumeinheiten (CTUs) ein. Liste der gültigen Werte.

video_codec_profile: legt das Kodierungsprofil fest. Die derzeit unterstützten HEVC-Profile sind: main, main10, main12, main422-10, main422-12, main444-8, main444-10, main444-12. Standard: main.

video_reference_frames:: Begrenzt die Anzahl der Referenzrahmen für HEVC. Standard: 3.

video_bframes:: Begrenzt die Anzahl aufeinanderfolgender bframes, die für HEVC verwendet werden. Standard: 3.

tuning:: Optimiert das ausgegebene Video, um es für eine bestimmte Codierungspriorität zu optimieren. Beachten Sie, dass für HEVC Video, die zulässigen Werte für tuning sind:

  • korn - optimiert für Filme mit hohem Korngehalt.
  • psnr - verwendet das "Spitzensignal-Rausch-Verhältnis", um die Videoqualität zu optimieren.
  • ssim - verwendet "strukturelle Ähnlichkeit", um die Videoqualität zu optimieren.
  • fastdecode - reduziert die Komplexität der Codierung, um eine einfachere Dekodierung zu ermöglichen.
  • Nullität - x265 behält einen internen Framespuffer bei, um die Qualität zu verbessern. Diese Einstellung entfernt diesen Puffer, reduziert jedoch die Qualität.

crf:: CRF ist eine Bitratensteuerungseinstellung und legt einen konstanten Ratenfaktor fest. Die Einstellung Zencoder-Qualität bestimmt eine geeignete CRF für ein bestimmtes Video. Wenn Sie also sowohl Qualität als auch CRF festlegen, überschreibt die CRF-Einstellung die Qualität. Gültige Werte sind 0-51, wobei niedrigere Werte eine bessere Qualität bedeuten. Im Allgemeinen sollten Sie nicht unter 16 gehen müssen; 16 ist für die meisten Dateien nahezu verlustfrei. Etwa 24 sieht normalerweise ziemlich gut aus. Etwa 35 sieht ziemlich komprimiert aus.

Stichprobe: HEVC + MP4

Im Folgenden finden Sie Beispieleinstellungen für einen Job, der eine HEVC-Ausgabe (als mp4 muxed) mit einer zusätzlichen MP4-Ausgabe für Clients erstellt, die HVEC noch nicht unterstützen.

{
  "input": "https://user:pass@bucket-name/file-name.mov",
  "outputs": [
      {
          "label": "hevc",
          "url": "s3://output-bucket/h265.mp4",
          "video_codec": "hevc",
          "video_codec_profile": "main",
          "audio_codec": "aac",
          "format": "mp4",
          "size": "1920x1080"
      },
      {
          "label": "h264",
          "url": "s3://output-bucket/h264.mp4",
          "video_codec": "h264",
          "video_codec_profile": "main",
          "audio_codec": "aac",
          "format": "mp4",
          "size": "640x360"
      }
  ]
}
