FAQ: Dateiübertragungen

In diesem Thema werden Fragen zum Übertragen von Dateien an Zencoder zur Codierung und an Ihre Ziele für Ausgaben beantwortet.

FAQ

  • Welche Übertragungsprotokolle unterstützen Sie? HTTP? FTP?

    Eingabe: Wir unterstützen HTTP, HTTPS, FTP, SFTP, Rackspace-Cloud-Dateien, Google Cloud Storage, Azure und Amazon S3.

    Ausgabe: Wir unterstützen FTP, SFTP, Rackspace-Cloud-Dateien, Azure, Google Cloud Storage und Amazon S3. Wir unterstützen derzeit keine allgemeinen HTTP-Uploads.

    Benutzer von Cloud-Dateien: Sie können die zu verwendende Region (DFW, ORD oder UK) angeben, indem Sie sie dem Protokoll hinzufügen, cf+ord://username:api_key@container/objectz. Derzeit werden nur DFW, ORD und UK als Cloud Files-Regionen unterstützt. Die DFW-Region wird standardmäßig verwendet.

    Benötigen Sie eine andere Übertragungsmethode? Lass uns wissen.

  • Wo kann ich meine Originaldateien zum Transcodieren ablegen?

    Wir können Dateien von überall über HTTP, HTTPS, FTP oder SFTP abrufen. Wir können Dateien auch aus einem Azure-Container, Google Cloud Storage oder S3-Bucket, einschließlich privater Buckets, und aus Rackspace Cloud-Dateien abrufen.

    Beispiele:

    http://example.com/path/to/file.avi
http://s3.amazonaws.com/bucket/file.avi
s3://bucket/file.avi
gcs://bucket/file.avi
cf://username:api_key@container/file.avi
azure://account-name:account-key@container/file.avi
https://example.com/path/to/file.avi
ftp://example.com/path/to/file.avi
sftp://example.com/path/to/file.avi
    Hinweis zu Ports: Wenn Sie einen Port in die Dateiadresse aufnehmen müssen, muss dieser am Ende der Domäne in der URL angegeben werden, wie folgt: 
    sftp://example.com:8888/path/to/file.avi

  • Können Sie Dateien sicher übertragen?

    Sicher. Verwenden Sie SFTP oder HTTPS und fügen Sie der URL einen Benutzernamen und ein Kennwort hinzu:

    
https://user:pass@example.com/file.mp4
sftp://user:pass@example.com/file.mp4

  • Können Sie Dateien aus einem privaten S3-, Google Cloud Storage- oder Cloud Files-Bucket abrufen?

    Ja. Sie können Zugriff hinzufügen Referenzen auf Ihr Zencoder-Konto oder Gewähren Sie Zencoder Lesezugriff auf den Bucket.

  • Gibt es Dateigrößenbeschränkungen?

    Nein. Wir haben keine Dateigrößenbeschränkungen. Tatsächlich haben wir uns dafür entschieden, keine großen Gebühren für große Eingabevideos zu erheben, um Kunden zu ermutigen, uns ihre besten Kopien zu senden. Je höher die Qualität der von Ihnen gesendeten Videos ist, desto höher ist die Ausgabequalität, die wir Ihnen bieten können.

    Es gibt jedoch die Dauer ist auf 24 Stunden begrenzt des Quellvideos.

  • Wohin gehen die fertigen Videos?

    Sie haben einige Möglichkeiten, wohin wir die Videos senden.

    Wir können die Dateien an einen FTP- oder SFTP-Server, an einen S3- oder Google Cloud Storage-Bucket oder an Rackspace Cloud Files senden. Siehe die URL-Option in unseren API-Dokumenten Für mehr Information.

    Vergiss nicht hinzuzufügen Referenzen oder Geben Sie Zencoder Zugriff auf Ihren Eimer Wenn Sie S3 oder Google Cloud Storage verwenden.

    Oder Sie können kein Ausgabeziel angeben, und wir behalten die Datei 24 Stunden lang bei. Wir stellen Ihnen eine URL zur Verfügung, über die Sie das Video herunterladen können. Wir werden das Video nach 24 Stunden entfernen. Vergessen Sie also nicht, es herunterzuladen.

    Wenn wir ab API-Version 2 keine Dateien auf Ihren Server hochladen können, laden wir sie stattdessen in S3 hoch und geben Ihnen eine URL für alle Dateien.

  • Wie lege ich die ACL-Berechtigungen fest, wenn ich verschlüsselte Dateien an S3 sende?

    Zencoder läuft auf Amazon EC2-Instanzen, daher gibt es eine Vielzahl von IP-Adressen, die wir für die Datei- und Live-Transcodierung verwenden. Amazon fügt der Liste regelmäßig neue Bereiche hinzu, die über AWS abgerufen werden können IP-Bereichs-API.

    Benachrichtigungen

    Derzeit werden Benachrichtigungen von gesendet sg-77f03012, dies kann sich jedoch jederzeit ändern. Um dies zu berücksichtigen, haben wir eine hinzugefügt X-Zencoder-Notification-Secret Header zu allen gelieferten Benachrichtigungen. Dies kann auf der gefunden werden API Dashboard, in dem Sie Ihr aktuelles Geheimnis sehen und bei Bedarf ein neues generieren können.

    Dieser Header wird bei jeder Anfrage übermittelt und kann verwendet werden, um sicherzustellen, dass die Benachrichtigung von Ihrem Zencoder-Konto übermittelt wurde. Hinweis:: Um Ihre Benachrichtigung geheim zu halten, stellen Sie sicher, dass Sie Benachrichtigungen an HTTPS-Endpunkte senden!

    Hinweis: Server können in neue Rechenzentren verschoben werden, und IP-Adressen können sich ohne vorherige Ankündigung ändern.

  • Kann ich eine Festplatte mit meinen Videos an Zencoder senden?

    Wenn Sie eine große Anzahl von Videos codieren möchten, diese aber nicht über das Internet übertragen möchten, können Sie Amazon verwenden AWS Import / Export. Sie senden eine Festplatte mit Ihren Videos an Amazon, das sie für Sie in S3 hochlädt.

    Amazon hat einfache Anweisungen um Ihr Laufwerk versandbereit zu machen, wohin es gesendet werden soll und vieles mehr.
